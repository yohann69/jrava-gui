Jrava: acceptance tests

The purpose of this document is to describe acceptance test scenarii, for the following user stories:

  1. application start/stop
    1. fresh starting application, with no storage
    2. closing application
      1. with standard window closing button (top-left corner)
      2. with Exit button
  2. activities
    1. starting the creation of an activity
    2. aborting the creation of an activity
      1. using standard window closing button
      2. using Cancel button
    3. creating a valid activity
    4. creating an invalid activity
      1. where title is missing
      2. where date is missing
      3. where duration is missing
      4. where distance is missing
      5. where date is invalid
      6. where duration is invalid
      7. where distance is invalid
    5. viewing activity list
      1. when there is no activity
      2. when there is at least an activity
    6. viewing activity detail
    7. updating activity
      1. title
      2. description
      3. both title and description
  3. viewing statistics
    1. when there is no activity recorded
    2. when there are several activities (several kinds, several dates) for current month/year

Fresh starting application (no storage use)

Preconditions

  • Application isn't started.

Actions

  1. Application (GraphicalMmiNoStorageMain) is started, with no argument.
    • N.B. application is started directly from IntelliJ, working directory is project root

Expected behaviour

  • The following message is printed on console output:
    • Jrava [graphical mode, no storage] starting...
  • Application's main frame is displayed, showing main menu with no existing activity.

Closing application, with standard window closing button

Preconditions

  • Application is started, displaying main frame.

Actions

  1. Standard window closing button (top-left corner) is clicked.

Expected behaviour

List activities when there is no activity recorded

Preconditions

  • Application is fresh started, displaying main menu.

Actions

  1. List activities button is clicked.
  2. Standard window closing button is clicked.

Expected behaviour

  • Step 1:

    • activity list frame is displayed, showing no activity.

  • Step 2:

    • main frame is displayed, showing no activity.

List activities when there is several activities recorded

Preconditions

  • Application is started, displaying main menu
  • At least 11 activities have been created, at least one of each type.

Actions

  1. List activities button is clicked
  2. RUN is selected as filter
  3. SWIM is selected as filter
  4. RIDE is selected as filter
  5. <None> is selected as filter
  6. standard window closing button is clicked

Expected behaviour

  • Step 1:
    • activity list frame is displayed, showing all activities but only 10 are visible (the last one can be seen by scrolling down).

  • Step 2:
    • activity list frame is displayed, showing only RUN activities.
  • Step 3:
    • activity list frame is displayed, showing only SWIM activities.
  • Step 4:
    • activity list frame is displayed, showing only RIDE activities.
  • Step 5:
    • activity list frame is displayed, showing all activities.
  • Step 6:
    • main frame is displayed, showing several activities.

    • activity detail frame is displayed, with updated title.

    • activity list frame is displayed, showing all activities, with updated title.

View statistics when there are several activities (several kinds, several dates)

Preconditions

  • Application is started, displaying main menu
  • One activity of each type has been recorded, and at least for each type
    • one of during current month
    • one during current year but not current month
    • one during neither current year neither current month
  • N.B. here, april 2022 is supposed to be current month

Actions

  1. View statistics button is clicked.
  2. RUN is selected as filter.
  3. SWIM is selected as filter.
  4. RIDE is selected as filter.
  5. <None> is selected as filter.
  6. Standard window closing button is clicked.

Expected behaviour

  • Step 1:
    • stats frame is displayed, showing non zero stats values.

  • Step 2:
    • stats frame is displayed, showing only RUN activities.

  • Step 3:
    • stats frame is displayed, showing only SWIM activities.

  • Step 4:
    • stats frame is displayed, showing only RIDE activities.

  • Step 5:
    • stats_ frame is displayed, showing same content as Step 1.

  • Step 6:
    • main frame is displayed, showing several activities.
